## 什么是VPS远程桌面连接？

简单说，远程桌面协议（RDP）是微软开发的一套东西，让你可以通过网络，像坐在那台电脑前面一样操控远端的机器。

你在家打开笔记本，连上一台部署在美国或者香港机房的Windows VPS，然后看到的就是那台服务器的完整桌面——浏览器、任务管理器、文件资源管理器，全都在。鼠标点哪，那边就响应。

这有什么用？用途其实很多：

- **外贸业务**：用海外IP挂着各种外贸工具，不影响本地网络
- **程序开发**：在高性能的远程服务器上跑编译任务，本地电脑只负责看结果
- **量化交易**：交易机器人需要7×24小时不间断运行，放本地电脑不现实
- **科学上网**：用海外IP环境做数据采集或账号管理
- **游戏加速**：某些网游需要特定地区的低延迟环境

RDP本身支持多显示器、剪贴板共享、本地打印机映射、文件传输，用起来相当流畅，不像某些远程工具那样有明显的画面撕裂感。

## 如何进行VPS远程桌面连接：分系统详细步骤

买好VPS、服务商发来IP地址和密码之后，接下来就是建立远程桌面连接了。

### Windows电脑连接Windows VPS（最常见场景）

这是最顺手的情况，Windows系统自带了"远程桌面连接"工具（mstsc），不需要装任何第三方软件。

**第一步：打开远程桌面连接**

在任务栏搜索框输入"远程桌面连接"或者"mstsc"，点击打开。也可以按 `Win + R`，输入 `mstsc` 回车。

**第二步：输入VPS的IP地址**

在"计算机"一栏填入VPS的IP地址。如果服务商告诉你RDP端口不是默认的3389，那就填 `IP地址:端口号`，例如 `123.456.789.0:1234`。

**第三步：输入用户名和密码**

点击"连接"，会弹出窗口让你输入凭据。用户名通常是 `Administrator`，密码是服务商提供的初始密码。

**第四步：接受证书提示**

第一次连接会弹出SSL证书警告，勾选"不再询问是否连接到此计算机"，然后点"是"继续。

等几秒钟，远程桌面就打开了。顺利的话，你会看到Windows Server的桌面界面出现在你眼前。

---

### Mac电脑连接Windows VPS

Mac没有内置RDP客户端，但微软官方在App Store提供了免费的 **Microsoft Remote Desktop** 应用，下载安装就行。

打开之后点击右上角的"+"添加新连接，在 **PC name** 一栏填入VPS的IP地址（如有非标准端口，格式同样是 `IP:端口`），然后填写用户名和密码，保存之后双击启动连接即可。

## 如果VPS的Windows系统没有开启RDP怎么办？

有一种情况：服务商给你的Windows VPS，RDP默认是关闭状态的。这时候需要先通过VNC（通常控制面板里有）登录进去，手动开启RDP。

步骤是这样的：

1. 通过控制面板的VNC进入Windows系统
2. 右键"此电脑"→"属性"→"远程设置"
3. 在"远程"选项卡里，勾选"允许远程连接到此计算机"
4. 建议同时勾选"仅允许运行使用网络级别身份验证(NLA)的远程桌面连接"，安全性更高
5. 进入防火墙设置，确认3389端口的入站规则是开放状态

设置完成之后，再用mstsc连接就可以了。

Evoxt的Windows VPS通常部署完成后RDP就是可用状态，这个步骤一般不需要，但知道原理总是有备无患。

## VPS远程桌面连接常见问题解答

**Q：连接时提示"无法连接到远程计算机"怎么办？**

首先检查IP地址和端口有没有填错。其次，确认VPS的防火墙没有封锁3389端口。如果两样都没问题，可以试试在服务商控制台重启一下VPS。

**Q：连接上去之后画面卡顿、延迟高怎么处理？**

通常是两个原因：一是VPS机房离你太远，网络延迟本来就高；二是远程桌面的画质设置太高占用了带宽。

可以在"远程桌面连接"的"显示"选项卡降低颜色深度，在"体验"选项卡里关掉壁纸、视觉特效等选项。画面会变得朴素，但流畅度会明显提升。

如果是延迟问题，选择离你更近的机房节点能根本性地解决。比如国内用户，选Evoxt的香港或马来西亚节点，延迟会比欧美节点低很多。

**Q：Windows VPS需要安装额外的软件才能用RDP吗？**

不需要。Windows系统本身就内置了RDP服务，不需要任何额外安装。你只需要在连接端有RDP客户端（Windows自带mstsc，Mac下载Microsoft Remote Desktop）就够了。
